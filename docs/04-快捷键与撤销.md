# 快捷键与撤销机制

## 快捷键系统

Flutter 原生三层架构：`Shortcuts → Actions → Intent`。

三层：
- **全局层**（`GlobalShortcutsWrapper`）：Esc 返回，独立于 AppShell，所有页面都生效
- **页面层**：导读页 F5 刷新 / Ctrl+Tab 切换
- **编辑器层**：动态生成，工具栏快捷键与 `toolbar_config.dart` 单一数据源绑定

快捷键序列化：字符串 `"Ctrl+T"` ↔ `SingleActivator`，由 `ShortcutHelper.parse(keyStr)` 双向转换。

## Esc 返回逻辑

Esc 和 Android 返回键走不同路径但共享通用后退逻辑：

```
Esc → Shortcuts → GoBackIntent → _handleEscBack()
Android 返回 → didPopRoute() → _handleAndroidBack()
```

区别：Esc 只做栈后退（Navigator.pop → GoRouter.pop），不做 tab 切换或双击退出。

## 编辑器撤销/重做

不自己实现。全交给 Flutter 的 `UndoHistory`（3.16+），它自动监听 `TextEditingController.value` 的 `notifyListeners()`：

- BBCode 操作通过 `controller.value = newValue` 修改 → 自动入 UndoHistory 栈
- 工具栏的 `UndoHistoryController` 用于按钮灰显
- `Ctrl+Z` 不拦截，自然冒泡到 `EditableText` 的 UndoHistory
- 不拦截 `Ctrl+Y` / `Ctrl+Shift+Z` 重做
