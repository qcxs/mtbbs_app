# 编辑器

## 五种模式

发帖 / 评论 / 回复评论 / 编辑帖子 / 编辑评论，通过 `EditorType` 枚举区分，URL 绑定见 `EditorPage._buildUrl()`。

## 数据流

```
PageFetcher 提取 formhash/hash/uid → 编辑器初始化
  → 用户编辑 → 提交 → API（只做纯提交，不做 fetch）
  → parseSubmitResponse() 统一解析 → SnackBar 结果
```

API 层重构后就地：旧架构 API 自己做 fetch，新架构 API 只提交，formhash 由编辑器统一传入。

## 快照系统

编辑器内容自动保存到 SharedPreferences，sessionKey 生成规则：`editor_snapshot_{host}_{tid}?_${pid}?_${replyPid}`，formhash 过期后快照仍有意义（发帖场景）。

自动保存去重：`contentHash` 比较，去重窗口 2 秒。

提交后保留快照（不自动删除），用于 "再次编辑" 场景。

## 三种图片来源

1. 上传的（`_handleImageUpload` → 调用 `/misc.php?mod=swfupload`）
2. 未使用的（`upload_api.fetchUnusedImages`）
3. 已绑定的（编辑帖子时已有的附件）

`ImagePickerSheet` 统一管理，选中后插入 `[attachimg]aid[/attachimg]`。

## 表情

在 `EmojiLoader` 初始化时将 `[s:{id}]` 及 Smilies 等预处理为 `{text}{emoji_{id}}{text}` 格式，便于后续 BBCode 解析。

## MT 图床

`MtImageHosting` 懒加载服务：首次上传时 GET `img.binmt.cc` → 从 HTML `<meta name="csrf-token">` 提取 token → 缓存 session 约 2 小时。
上传队列支持多文件，历史记录持久化到 SharedPreferences（最多 200 条，编辑器只显示最近 20 条未隐藏的）。

## 粘贴图片

Ctrl+V 拦截：PowerShell `Get-Clipboard` 检测剪贴板图片 → 保存临时文件 → `upload_api.uploadImage()` 上传 → 插入 `[attachimg]`。无对话框，无文本回退。
