# 状态管理与站点配置

## 状态管理

`ChangeNotifier + Provider`。`notifyListeners()` 统一由持久化基座方法调用。

关键 Provider：`SettingsProvider`（全局设置）、`AuthProvider`（登录态）、`HistoryProvider`（浏览记录）。

## 站点切换

严格顺序不可颠倒：

1. `saveCurrentSiteState()` — 持久化当前站点的状态（如编辑中的 formhash）
2. `SiteConfig.switchTo(index)` — 更新 `SiteConfig.current` 和 `ApiService` 的 baseUrl
3. 各 Provider 重新 `load()` 对应站点的数据

## 持久化

- SharedPreferences key 命名：`{configKey}_{host}`，按站点 host 隔离
- Cookie 存储路径：`{docDir}/cookies/{host}/{accountName}`
- 缓存策略：API 数据用 SharedPreferences，图片用 `flutter_cache_manager`

## Dio 拦截器链

```
CookieManager → InterceptorsWrapper（全局错误处理） → LogInterceptor
```

CookieManager 使用 `PersistCookieJar`，按 host + 账号隔离。
