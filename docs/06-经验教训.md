# 经验教训

## 解析相关

1. **Discuz inajax 的 content-type** — 只有少量页面返回 XML，多数返回 HTML（"discuz_uid" 标记），还有 JSON 或纯文本。**不要假设 content-type**，以响应内容为准。
2. **优先 DOM 解析**（`xml_helper.dart`）—— 即使 HTML 不规范也优先 DOM，正则只用于 DOM 无法处理的场景（如行内表情、签名混合内容）。
3. **`parseSubmitResponse` 统一入口** — 所有表单提交的响应解析走此函数，它处理 `succeed`、`alert`、`location` 等 Discuz 通用返回格式。

## 状态与导航

4. **`Navigator.maybePop` vs `canPop + pop`** — `canPop` 检查 `Navigator`，非 GoRouter。在 GoRouter shell 环境下，`Navigator.canPop` 可能返回 false 但 `GoRouter.canPop()` true。
5. **`showModalBottomSheet` 数据同步** — 关闭重开会重建 State，用 `ValueNotifier` / 回调而非内部 State 保持数据同步。
6. **双击退出** — 只在 Android 返回键触发，Esc 不做。
7. **站点切换顺序** — `saveCurrentSiteState()` → `SiteConfig.switchTo()` → Provider reload，不可颠倒。

## 编辑器

8. **撤销不自己实现** — Flutter UndoHistory（3.16+）代码量为 0，Bug 风险低。所有 BBCode 操作通过 `controller.value = newValue` 修改即可自动入栈。
9. **工具栏单一数据源** — `toolbar_config.dart` 控制所有项的枚举、默认顺序、快捷键、显隐。`BBCodeToolbar`、`EditorPage`、设置页都从此派生。
10. **工具栏切换为"只包裹+清除样式"** — 不再做"选中后切换"（如已有 `[b]` 时取消加粗），用户感受类似 Word 的 toggle。取消加粗只能通过 Undo 或 ClearStyles。
11. **编辑器不拦截 `Ctrl+Z`** — 让 `EditableText` 的 UndoHistory 自然处理。

## 图片

12. **图片查看器 → photo_view** — v3 版改用 `photo_view` 包，解决手势冲突和缩放动画，之前的两版（自己实现）已废弃。
13. **图片上传用独立 Dio** — CDT（Content Delivery Network）不兼容论坛 Dio 的请求头，图片缓存走 `IgnoreCacheFileService`（自定义 30 天/7 天有效期），不用 `ApiService().dio` 下载。
14. **MIUI 相册自动保存已选图片（已知特性）** — 部分国产 ROM（已确认的小米 MIUI/HyperOS）的系统相册在处理 `ACTION_GET_CONTENT`/PhotoPicker 等系统 Intent 时，会自动将选中的图片保存到相册。此行为由 OEM 系统层控制，Flutter 应用层无法干预。模拟器（AOSP 原生相册）不复现。非功能缺陷，无需修复。如要彻底避免，需改用 `photo_manager` 直接查询 MediaStore 绕过系统相册，但会牺牲 Windows 等桌面平台的图片选取能力。

## 通用

17. **MCP 工具优先** — 新增 API 时先用 Chrome MCP 确认请求/响应格式，再写 Dart 代码。
18. **日志强制 AppLogger** — 严禁 `print`/`debugPrint`，已配置 lint 规则 `no_printf` 阻止。
19. **Controller 绑定优于回调** — 跨 Widget 状态传递用统一的 Controller 实例（如 `BBCodeToolbarController`），比多层回调更易维护。
