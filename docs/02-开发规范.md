# 开发规范

## API 新增流程（8 步）

1. 分析目标页面 → 确定 URL 和参数
2. `lib/api/{name}/` 下创建三个文件
3. `http.dart` — 发送请求，只做 `dio.get/post`，**不允许**调 parse
4. `parse.dart` — 纯函数解析 HTML/XML → JSON，**不包含**网络逻辑
5. `test/api/{name}/` 下创建 `{name}_test.dart` — CLI 测试入口，接收 `--base-url` 参数
6. `export.dart` — 导出 `fetchXxx` 便捷方法
7. 挂载到 Flutter 服务层（`services/` 或页面层调 export）
8. 验证

## 文件命名规范

- `lib/api/{name}/` — 三文件：`http.dart` / `parse.dart` / `export.dart`
- `test/api/{name}/` — 测试文件：`{name}_test.dart`
- `lib/services/` — `{domain}_service.dart`（如 `mt_image_hosting.dart`）
- `lib/pages/{feature}/` — 页面 + 子组件拆分，>500 行时拆分
- `lib/widgets/` — 功能独立的可复用组件

## 日志规范

- 全局：`AppLogger.{d/i/w/e}(tag, message)`，**禁止** `print` / `debugPrint`
- 标签约定：`DIO` / `PARSE` / `PAGE` / `CACHE`
- 开关：`AppLogger.enabled = false` 全局关闭

## 通用 UI 规范

- 对话框：`maxWidth: 360~420`，必须有关闭按钮，`barrierDismissible: true`
- 错误页：`PageErrorWidget` 统一使用，不弹 SnackBar，提供重试按钮
- 站点切换类界面不显示关闭按钮（用户必须选一个站点）
- 点赞/收藏等无参数操作：直接 GET，无对话框
